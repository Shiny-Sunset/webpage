FingerGuider

鋭意制作中!! (チーム開発)

～作品紹介～

マウスやカメラに映した人差し指で線を引いて、画面上の3Dダンゴムシとインタラクションできるアプリです。
線が近づくとダンゴムシが逃げ出し、触れると丸まり、線が消えるとまた歩き出します。
ダンゴムシの数は1〜10匹まで調整でき、ダンゴムシ同士がぶつかると互いに避け合います。

カメラを使った指モードでは、人差し指を立てると線が引かれ、
他のジェスチャー（グー・パーなど）に切り替えると線が止まります。

～制作の話～

大学の授業の課題として、Seia1004さんと2人で開発しています。
技術スタックはVite + Vanilla JavaScript + Three.js + Canvas 2D APIで、
カメラによるジェスチャー認識にはMediaPipe Tasks Visionを使用しています。

私はアプリの企画・設計と、ダンゴムシの3Dモデル制作を担当しました。

現在はPhase 2（MediaPipe連携・3Dモデル化・複数匹対応・衝突回避）まで完了しており、
次のPhase 3ではダンゴムシをちびキャラモデルに置き換え、感情表現を追加する予定です。

Phase 3に向けて、ちびキャラのモデリング・リギング・スキニング・テクスチャ作成・
アニメーション作成・シェイプキー作成まではすでに完了しています。
今後はプログラム側の実装にも着手していきたいと考えています。

～今後の展望～

現在はチームメイトに大まかなプロトタイプを作ってもらった段階で、
ここから細部のこだわりを私が実装していく予定です。

まず、ダンゴムシをちびキャラに切り替えられる機能を実装します。
次に、ダンゴムシの特徴である交替性転向反応のロジックを組み込み、
動きをよりリアルに近づけていきます。
また、餌やりなどインタラクションの種類も増やしたいと考えています。

さらに、現在のメインである線を引くシミュレーション機能に加えて、
カメラを活用した新しい機能も計画中です。
教授にプロトタイプを見せたところ「カメラで人の顔にダンゴムシが寄ってくると面白い」
という話になり、それをきっかけに構想が広がりました。

カメラで写真を撮るとちびキャラのカットイン演出が表示され、
人の顔にダンゴムシが寄ってきて、紙を食べるというダンゴムシの特徴を活かして
顔の部分だけを食べてしまう――そんな虫食いシミュレーションまで
作り上げることを目標にしています。

虫食いが終わった後には、食べた穴からダンゴムシが顔をのぞかせたり、
画面の端でちびキャラがドヤ顔とピースで決めポーズをとる演出も予定しています。
完成した虫食い画像は自由にダウンロードできるようにして、
シミュレーションの結果をSNSなどでシェアして楽しめるようにしたいと考えています。
